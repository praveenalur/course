Business Case 2: Sorting Public Transports

Problem Statement

Sort a list of transportation services based on their daily ridership in ascending order for better analysis and decision-making.

Challenges

  • Representing transportation data effectively.
  • Implementing an efficient sorting mechanism.
  • Handling merging of subarrays correctly.
  • Ensuring scalability for larger datasets.

Solutions

  • Use the Transport structure to store name and daily ridership.
  • Employ the Merge Sort algorithm for efficient sorting (O(nlogn)).
  • Divide the logic into merge and mergeSort functions for clarity.
  • Use temporary arrays to facilitate stable merging.

Algorithm

  1. Divide: Split the array into two halves recursively.
  2. Conquer: Merge sorted halves by comparing elements.
  3. Combine: Reconstruct the original array in sorted order.

Steps

  1. Define the Transport structure.
  2. Initialize transport data in an array.
  3. Use mergeSort to recursively sort the array.
  4. Merge sorted parts using the merge function.
  5. Display the sorted results.

Conclusion

The program sorts transportation services efficiently using Merge Sort, ensuring accuracy and scalability for larger datasets.
